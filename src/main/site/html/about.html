Call functions in one script, from another

Separation of concerns is a keystone of good software design. But that's hard when your entire application has to be in one file - all your functions, all your state, all your variables. It would be great if, when appropriate, you could split your applications up into different scripts. Now, you can.

call-other-script provides a boilerplate that you paste into the top of your scripts to either make some of their functions available to other scripts (as a library) or to give them access to the functions in library scripts created this way. 

string callOtherScript(string scriptName, string functionName, string jsonArguments)

string
scriptName
The name of the script that contains the function you want to call.
string
functionName
The name of the function you want to call.
string
jsonArguments
The arguments to pass to the target function, in a JSON string.

your_script.lsl 
default
{
	state_entry
	{
		string result = callOtherScript( "library.lsl", "some_function", "the input" );
		llOwnerSay( "The other script said '" + result + "'!" );
	}
}
